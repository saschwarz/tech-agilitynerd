Adding pyrsvg to a virtualenv created with --no-site-packages

Tue 12 October 2010
By Steve Schwarz

In python.

tags: pythonrsvgvirtualenv

I set up my development and deployment environments on Ubuntu with virtualenvwith the --no-site-packages option to isolate them from packages in the system installation. My application uses pyrsvgand it is installed by default as a system package. Consequently I had to link the shared libraries it installs (w/in gtk) into my virtualenv.

Here are the links I created (workon and cdsitepackages are virtualenvwrappershell aliases):

$ workon project
$ cdsitepackages
$ ln -s /var/lib/python-support/python2.6/gtk-2.0/rsvg.so .
$ ln -s /var/lib/python-support/python2.6/gtk-2.0/gobject .
$ ln -s /var/lib/python-support/python2.6/gtk-2.0/glib .
